Modules:

Broker: Allows information to be stored and consumed by the various modules in the project.
Broker Topics:
"moodprismTopic" -> Raw data generated by the client. This data is consumed by MoodPrism and presented directly in the dashboard.
"moodTopic" -> Data sent to be consumed by the moodApp. The moodApp then consumes the data and assings to each user a mood classification. This mood is then sent to the broker with the topic "moodPrismTopic".
Moodprism: Application that consumes all the data stored in the broker and sends it in JSON format to a web application.
MoodApp: Application that consumes data from the broker and assigns a mood classification. This classification is then inserted into the broker.
ClientApp: Application inicitialized on the client size, sends the information obtained through the keyboard and mouse to the broker. When starting, this application acepts two arguments, the first being the username and the second being the ip to where the information is being sent.
FrontEnd: The front-end is built with React, that allows the injection of the information obtained in the back-end, and is dynamic, meaning that it is capable of handling a different number of users at any given time.
How to access: The web application with just JSON information can be accessed on 192.168.160.103:10080. The web application with information organized through React can be accessed on 192.168.160.103:1070.

Logging:

For the logging of our microservices we use the logging framework Logback and the ELK Stack. The Logback framework allows us to send logs about kafka messages directly to the Logstack via configuration file with a Logback appender. The Logstash then process the JSON Log Data and sends it to the ElasticSearch that, with Kibana, allows us to visualize statistical data about the kafka messages that were sent over time in a chronological order. We can also track how many messages are exchanging in our system and therefore analyze its performance.

Jenkins Pipeline:

Stage Build: Used to generate jar file.
Stage Test: Used to start the tests created. (Tests are not 100% funcitional)
Stage Deploy: Used t send the jar to the Artyfactory.
Stage Build Docker Image: Used to generate Docker Image, tag and push it to the Registry.
Stage Deploy RunTime: Used to send the image to the VM.
